Clustering: Hierarchical and Partition-based

As with previous weeks, the working environment for this week's practicals is a set of Jupyter notebooks.

Each of these notebooks should be downloaded as a text file and run using jupyter-lab.

The notebooks can be obtained from the following links:

  • Using a variety of clustering techniques on blob data
  • Use of GMM (soft) clustering
  • Using k-means to cluster generated blob data, while tracking inertia and silhouette scores
  • Comparing linkage functions on a simple dataset, using the AGNES hierarchical clustering algorithm
  • Using k-modes clustering on a small set of notional student grades
  • Using k-modes and k-prototypes clustering on MPG data used in R's ggplot2 demos
  • Visualising PCA, t-SNE and UMAP for visualising "clustered" audio data

You should right-click and download each file to the same directory, and then follow the advice below.

Each notebook is extensively annotated.

You should study each step in the notebook and to note any interesting behaviour, as this should help you gain a better understanding of the concepts frpm the lecture.

Students should also download the following data files:

  • Download this data file and save in a data/ folder under the folder where Practical_A_KM_GMM.ipynb was saved.
  • Download this data file and save in a data/ folder under the folder where Practical_F_studentGrades_Kmodes.ipynb was saved.
  • Download this data file and save in a data/ folder under the folder where Practical_G_AutoMPG_KmodesPrototypes.ipynb was saved.
  • Download this data file and save in a data/ folder under the folder where Practical_H_Isolet_Visualisation.ipynb was saved.

Installation

You will need to update your python packages by installing hdbscan and python-graphviz, using the following commands:

1
2
3
conda install -y conda-forge::hdbscan

conda install -y conda-forge::python-graphviz

You might also need to install ipynbname:

1
conda install -y conda-forge::ipynbname

For Practical_H_Isolet_Visualisation.ipynb, you will also need to install umap-learn:

1
conda install -y conda-forge::umap-learn

Unfortunately, umap-learn is not currently (March 2025) consistent with the latest version of numpy, which is v2.x.

Users need to downgrade numpy to v1.x instead.

The way I did this was to create a new (and additional) environment based on python 3.11 instead of python 3.12. By activating this environment instead, it was possible to run the notebook with UMAP.
